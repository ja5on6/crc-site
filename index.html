Introduction

I recently came across The Cloud Resume Challenge by Forrest Brazeal and thought it's a pretty cool side project. To have the certifications and knowledge is admirable, but to also do the hands on, build and debug, is respectable.

To go through my own "baptism by fire", I got started and created this completely serverless site.

It has a static frontend, a dynamic backend for counting the number of visitors (bottom of this page), and a DevOps pipeline to automate deployment.

4. Visitor counterðŸ‘‡

~ 20 hours

Now we come to building the backend of this site!

How this setup works is through a short javascript script on this page, querying the REST API endpoint on API Gateway. This triggers a Lambda function that updates and returns the visitor number from a DynamoDB table.

The beauty of this is at the scale of this site, this setup is practically free!

5. Back-end CI/CD with SAMðŸ‘‡

~ 20 hours

In step 4, while I assembled the back-end successfully using mainly point-and-click on the AWS management console (and pasting in my Python code in Lambda of cause), it will definitely be cooler if I can do all these automatically without logging in to the AWS console!

Never did I know it takes several software applications to be able to use SAM - namely AWS CLI, AWS SAM CLI, Python and Docker, and it's an learning experience to pick up all of them. Thankfully with SAM, it automates the following deployment steps: build, zip code, upload to S3, create role, create/update function, create REST API, create resource, create method, update integration and create table.

After some debugging, I finally managed to set up the back-end in an efficient CI/CD manner!
